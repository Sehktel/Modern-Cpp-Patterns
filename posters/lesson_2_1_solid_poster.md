## 🔄 L - Liskov Substitution Principle (LSP)

```
┌─────────────────────────────────────────────────────────────────┐
│              LISKOV SUBSTITUTION PRINCIPLE                     │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│  🔄 "ОБЪЕКТЫ ПРОИЗВОДНОГО КЛАССА ДОЛЖНЫ БЫТЬ ЗАМЕНЯЕМЫ         │
│     ОБЪЕКТАМИ БАЗОВОГО КЛАССА"                                 │
│  🎯 СОХРАНЕНИЕ ПОВЕДЕНИЯ ПРИ НАСЛЕДОВАНИИ                      │
│  🛡️ ГАРАНТИИ КОНТРАКТА БАЗОВОГО КЛАССА                       │
│  ⚠️ ПРЕДОТВРАЩЕНИЕ НЕОЖИДАННОГО ПОВЕДЕНИЯ                     │
└─────────────────────────────────────────────────────────────────┘

❌ НАРУШЕНИЕ LSP:
┌─────────────────────────────────────────────────────────────────┐
│                    ПЛОХОЙ ДИЗАЙН                              │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │                    Rectangle                                ││
│  │                                                             ││
│  │  • setWidth(int w)                                         ││
│  │  • setHeight(int h)                                        ││
│  │  • getArea() → width * height                              ││
│  │  • getPerimeter() → 2*(width + height)                    ││
│  └─────────────────────────────────────────────────────────────┘│
│                        ▲                                       │
│                        │                                       │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │                      Square                                 ││
│  │                                                             ││
│  │  • setWidth(int w)  → w, w (нарушение!)                    ││
│  │  • setHeight(int h) → h, h (нарушение!)                    ││
│  │  • getArea() → w * w                                       ││
│  │                                                             ││
│  │  ❌ Square НЕ МОЖЕТ ЗАМЕНИТЬ Rectangle!                    ││
│  │     (изменяет поведение setWidth/setHeight)                ││
│  └─────────────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────────────┘

✅ СОБЛЮДЕНИЕ LSP:
┌─────────────────────────────────────────────────────────────────┐
│                    ХОРОШИЙ ДИЗАЙН                             │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │                      Shape                                  ││
│  │                                                             ││
│  │  • getArea() → double                                      ││
│  │  • getPerimeter() → double                                 ││
│  │  • isValid() → bool                                        ││
│  └─────────────────────────────────────────────────────────────┘│
│                        ▲                                       │
│                        │                                       │
│  ┌─────────────────────┼─────────────────────┐                │
│  │                     │                     │                │
│  │  ┌─────────────┐    │    ┌─────────────┐  │                │
│  │  │  Rectangle  │    │    │   Square    │  │                │
│  │  │             │    │    │             │  │                │
│  │  │ • getArea() │    │    │ • getArea() │  │                │
│  │  │ • getPerimeter│    │    │ • getPerimeter│  │                │
│  │  │ • setWidth() │    │    │ • setSide() │  │                │
│  │  │ • setHeight()│    │    │ • isValid() │  │                │
│  │  └─────────────┘    │    └─────────────┘  │                │
│  │                     │                     │                │
│  │  ✅ Rectangle МОЖЕТ ЗАМЕНИТЬ Shape!      │                │
│  │  ✅ Square МОЖЕТ ЗАМЕНИТЬ Shape!         │                │
│  └─────────────────────┴─────────────────────┘                │
└─────────────────────────────────────────────────────────────────┘
```

## 💡 Практические рекомендации

```
┌─────────────────────────────────────────────────────────────────┐
│                  ПРАКТИЧЕСКИЕ РЕКОМЕНДАЦИИ                     │
└─────────────────────────────────────────────────────────────────┘

1. 🎯 SRP - Single Responsibility:
   • Один класс = одна ответственность
   • Одна причина для изменения
   • Высокая сплоченность
   • Легкое тестирование

2. 🔓 OCP - Open/Closed:
   • Открыто для расширения
   • Закрыто для модификации
   • Используйте абстракции
   • Применяйте Strategy pattern

3. 🔄 LSP - Liskov Substitution:
   • Подклассы заменяемы базовыми
   • Сохраняйте контракты
   • Не нарушайте поведение
   • Тестируйте заменяемость

4. 🎯 ISP - Interface Segregation:
   • Разделяйте большие интерфейсы
   • Специфичные интерфейсы
   • Избегайте принудительных зависимостей
   • Принцип "нужно, а не можешь"

5. 🔀 DIP - Dependency Inversion:
   • Зависите от абстракций
   • Используйте Dependency Injection
   • Инверсия контроля
   • Высокоуровневые модули независимы

6. 🧪 Тестирование:
   • SOLID код легко тестируется
   • Моки для абстракций
   • Изоляция компонентов
   • Unit тесты для каждого класса

7. 📚 Рефакторинг:
   • Постепенное применение SOLID
   • Выявляйте нарушения
   • Разделяйте ответственности
   • Вводите абстракции
```

   • L - Заменяемость подклассов

   • D - Зависимость от абстракций
