```

│                    OBSERVER PATTERN                            │

┌─────────────────────────────────────────────────────────────────┐

┌─────────────────┬───────────────────────────────────────────────┐

├─────────────────┼───────────────────────────────────────────────┤

```

## ✅ Решение с Observer

```
┌─────────────────────────────────────────────────────────────────┐
│                    РЕШЕНИЕ: OBSERVER PATTERN                  │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│                    ХОРОШИЙ ДИЗАЙН                             │
│                                                                 │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐  ┌─────────┐│
│  │   Observer  │  │   Observer  │  │   Observer  │  │ Observer││
│  │             │  │             │  │             │  │         ││
│  │ • update()  │  │ • update()  │  │ • update()  │  │ • update││
│  │ • display() │  │ • display() │  │ • display() │  │ • display││
│  └─────────────┘  └─────────────┘  └─────────────┘  └─────────┘│
│         ▲                 ▲                 ▲           ▲      │
│         │                 │                 │           │      │
│  ┌──────┴─────────────────┴─────────────────┴───────────┴──┐  │
│  │                    Subject                              │  │
│  │                                                         │  │
│  │  std::vector<Observer*> observers;                      │  │
│  │                                                         │  │
│  │  void attach(Observer* obs) {                           │  │
│  │      observers.push_back(obs);                          │  │
│  │  }                                                      │  │
│  │                                                         │  │
│  │  void detach(Observer* obs) {                           │  │
│  │      // Удаляем из списка                               │  │
│  │  }                                                      │  │
│  │                                                         │  │
│  │  void notify() {                                        │  │
│  │      for (auto* obs : observers) {                      │  │
│  │          obs->update();                                 │  │
│  │      }                                                  │  │
│  │  }                                                      │  │
│  └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│  Преимущества:                                                  │
│  ✅ Слабая связанность                                        │
│  ✅ Легко добавлять/удалять наблюдателей                      │
│  ✅ Соблюдение Open/Closed Principle                          │
│  ✅ Соблюдение Dependency Inversion Principle                 │
└─────────────────────────────────────────────────────────────────┘
```

## 🎨 Варианты реализации

### 1. Классическая реализация
```
┌─────────────────────────────────────────────────────────────────┐
│                  КЛАССИЧЕСКАЯ РЕАЛИЗАЦИЯ                      │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│ // Абстрактный наблюдатель                                     │
│ class Observer {                                               │
│ public:                                                         │
│     virtual ~Observer() = default;                             │
│     virtual void update(float temp, float humidity, float pressure) = 0;│
│ };                                                              │
│                                                                 │
│ // Конкретный наблюдатель                                       │
│ class CurrentConditionsDisplay : public Observer {             │
│ private:                                                        │
│     float temperature;                                          │
│     float humidity;                                             │
│                                                                 │
│ public:                                                         │
│     void update(float temp, float humidity, float pressure) override {│
│         this->temperature = temp;                               │
│         this->humidity = humidity;                              │
│         display();                                              │
│     }                                                           │
│                                                                 │
│     void display() {                                            │
│         std::cout << "Current conditions: "                    │
│                   << temperature << "°C, "                     │
│                   << humidity << "% humidity" << std::endl;    │
│     }                                                           │
│ };                                                              │
└─────────────────────────────────────────────────────────────────┘
```

### 2. Современная реализация с std::function
```
┌─────────────────────────────────────────────────────────────────┐
│              СОВРЕМЕННАЯ РЕАЛИЗАЦИЯ С STD::FUNCTION           │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│ class ModernSubject {                                          │
│ private:                                                        │
│     std::vector<std::function<void(float, float, float)>> observers;│
│     float temperature;                                          │
│     float humidity;                                            │
│     float pressure;                                            │
│                                                                 │
│ public:                                                         │
│     void attach(std::function<void(float, float, float)> obs) {│
│         observers.push_back(obs);                              │
│     }                                                           │
│                                                                 │
│     void detach(std::function<void(float, float, float)> obs) {│
│         // Удаляем observer                                     │
│     }                                                           │
│                                                                 │
│     void setMeasurements(float temp, float hum, float press) { │
│         temperature = temp;                                     │
│         humidity = hum;                                         │
│         pressure = press;                                       │
│         notify();                                               │
│     }                                                           │
│                                                                 │
│ private:                                                        │
│     void notify() {                                             │
│         for (const auto& observer : observers) {               │
│             observer(temperature, humidity, pressure);         │
│         }                                                       │
│     }                                                           │
│ };                                                              │
│                                                                 │
│ // Использование                                                │
│ ModernSubject weatherStation;                                  │
│                                                                 │
│ weatherStation.attach([](float t, float h, float p) {          │
│     std::cout << "Temperature: " << t << "°C" << std::endl;   │
│ });                                                             │
│                                                                 │
│ weatherStation.attach([](float t, float h, float p) {          │
│     std::cout << "Humidity: " << h << "%" << std::endl;       │
│ });                                                             │
└─────────────────────────────────────────────────────────────────┘
```

### 3. Реализация с weak_ptr
```
┌─────────────────────────────────────────────────────────────────┐
│              РЕАЛИЗАЦИЯ С WEAK_PTR                             │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│ class SafeSubject {                                            │
│ private:                                                        │
│     std::vector<std::weak_ptr<Observer>> observers;            │
│                                                                 │
│ public:                                                         │
│     void attach(std::shared_ptr<Observer> obs) {               │
│         observers.push_back(obs);                              │
│     }                                                           │
│                                                                 │
│     void detach(std::shared_ptr<Observer> obs) {               │
│         observers.erase(                                       │
│             std::remove_if(observers.begin(), observers.end(), │
│                 [&obs](const std::weak_ptr<Observer>& weak) {  │
│                     return weak.lock() == obs;                 │
│                 }), observers.end());                          │
│     }                                                           │
│                                                                 │
│     void notify() {                                             │
│         // Очищаем недействительные weak_ptr                   │
│         observers.erase(                                       │
│             std::remove_if(observers.begin(), observers.end(), │
│                 [](const std::weak_ptr<Observer>& weak) {      │
│                     return weak.expired();                     │
│                 }), observers.end());                          │
│                                                                 │
│         // Уведомляем активных наблюдателей                    │
│         for (auto& weak : observers) {                         │
│             if (auto obs = weak.lock()) {                      │
│                 obs->update(temperature, humidity, pressure);  │
│             }                                                   │
│         }                                                       │
│     }                                                           │
│ };                                                              │
└─────────────────────────────────────────────────────────────────┘
```

┌─────────────────────────────────────────────────────────────────┐

## 💡 Практические рекомендации

```
┌─────────────────────────────────────────────────────────────────┐
│                  ПРАКТИЧЕСКИЕ РЕКОМЕНДАЦИИ                     │
└─────────────────────────────────────────────────────────────────┘

1. 🎯 Выбор реализации:
   • Классическая - для простых случаев
   • std::function - для гибкости
   • weak_ptr - для безопасности
   • Event Bus - для сложных систем

2. 🔧 Принципы проектирования:
   • Слабая связанность между субъектом и наблюдателями
   • Абстракция через интерфейсы
   • Инкапсуляция логики уведомлений
   • Разделение ответственности

3. 🧪 Тестирование:
   • Используйте моки для изоляции тестов
   • Тестируйте регистрацию и отмену регистрации
   • Проверяйте корректность уведомлений
   • Тестируйте производительность

4. ⚠️ Типичные ошибки:
   • Циклические ссылки между субъектом и наблюдателями
   • Уведомления в конструкторе/деструкторе
   • Игнорирование исключений в update()
   • Неправильное управление временем жизни

5. 📚 Лучшие практики:
   • Используйте RAII для управления подписками
   • Применяйте weak_ptr для предотвращения циклов
   • Документируйте контракты Observer'ов
   • Рассмотрите асинхронные уведомления

6. 🚀 Оптимизация:
   • Кэшируйте список наблюдателей
   • Используйте move semantics
   • Рассмотрите batch уведомления
   • Профилируйте критические участки
```
